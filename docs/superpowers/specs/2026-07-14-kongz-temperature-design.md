# 콩즈 온도 설계

## 개요

`2026-07-14-attendee-rating-design.md`에서 정의한 참가 신청자 hover 카드가 현재 매너/실력/재만남 3개 평균을 그대로 노출한다. 이를 세 항목을 합산한 단일 "콩즈 온도"로 대체한다. 마이페이지(`mypage.js`)의 "내 정보" 3개 항목 노출은 변경하지 않는다.

## 계산식

```
가중평균 = 매너*0.25 + 실력*0.25 + 재만남*0.5   (각 1~5점)
콩즈 온도 = 36.5 + (가중평균 - 3) * 13
```

- 평균 3점(중립) → 36.5도, 평균 5점(만점) → 62.5도, 평균 1점(최저) → 10.5도
- `ratingCount`가 0이면 평균 대신 기본값 36.5도 반환 (신규 회원 취급)
- 소수 첫째자리로 반올림

`rating-logic.js`의 기존 `computeAverages(memberStats)`가 이미 sum/count를 계산하므로, 그 결과를 받아 온도로 변환하는 `computeKongzTemp(memberStats)` 함수를 추가한다. 별도 Firestore 필드나 마이그레이션 없음.

## 표시 (post.js 호버카드)

- 매너/실력/재만남 3줄을 제거하고 "콩즈 온도 36.5도" 한 줄로 대체
- 온도 숫자에 색상 적용 (인라인 style, 3단계 보간):
  - 10.5도: `#4a90d9` (파랑)
  - 36.5도: `#888888` (회색)
  - 62.5도: `#e74c3c` (빨강)
  - 10.5~36.5 구간, 36.5~62.5 구간 각각 선형 보간(RGB lerp). 범위를 벗어나면 양 끝 색으로 clamp.
- "N회 평가받음" 텍스트는 유지

## 범위 밖

- 마이페이지 3항목 노출 방식 변경 없음
- 온도 계산식/가중치를 관리자가 조정하는 UI 없음 (코드 상수로 고정)
